How Protected Routes Work

                    

                    

                        
// Frontend sends token in Authorization header
fetch('/profile', {
    headers: {
        'Authorization': 'Bearer eyJhbG...'  // ← Your JWT token
    }
});

// Backend @token_required decorator:
// 1. Extracts token from header
// 2. Verifies signature with SECRET_KEY
// 3. Checks if token expired
// 4. Stores user info in g.current_user
// 5. Calls the actual route function
